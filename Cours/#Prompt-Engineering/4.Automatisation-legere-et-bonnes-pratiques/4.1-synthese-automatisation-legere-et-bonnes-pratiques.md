
# Module 4 : Automatisation légère & bonnes pratiques -- Note de synthèse

## 1. Objectif du module

Utiliser l’IA pour automatiser des petites tâches répétitives (emails, rapports, classement de données) sans forcément coder.  
Le but : gagner du temps en construisant des prompts stables, réutilisables et faciles à adapter.

## 2. Idées clés à retenir

- Beaucoup de tâches “bêtes et répétitives” peuvent être déléguées à l’IA via un bon prompt.
- Un prompt d’automatisation doit décrire clairement l’entrée et le format de sortie attendu.
- Tu peux intégrer l’IA dans tes outils (Docs, Sheets, Notion…) comme une brique intermédiaire.
- L’automatisation reste “light” : c’est surtout une façon intelligente de réutiliser tes bons prompts.

## 3. Bonnes pratiques

- Commencer par une seule tâche précise et un format unique (ex : tableau markdown).
- Définir clairement ce qui doit être filtré, exclu ou transformé.
- Tester tes prompts avec plusieurs jeux de données avant de les “adopter”.
- Toujours garder un contrôle humain sur les premiers essais.

## 4. Prompts types à garder sous la main

```plain
[Modèle d’email]
Voici le contexte : [CONTEXTE]. 
Rédige un email [TYPE : relance, confirmation, info…] 
Ton : [TON]. Longueur : [COURT / MOYEN / LONG]. 
Structure claire avec objet, introduction, message principal, phrase de conclusion.

[Rapport à partir de données]
Voici des données brutes : 
[COLLER LES DONNÉES].
1) Résume les points principaux.
2) Fais ressortir 3 enseignements clés.
3) Propose une courte conclusion/action recommandée.

[Organisation de données]
Voici une liste d’éléments non triés : 
[LISTE].
Classe-les par catégories logiques, puis restitue-les dans un tableau markdown avec : catégorie, élément, commentaire utile.
```
