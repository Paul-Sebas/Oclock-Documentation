# Module 2 : Génération de contenu textuel -- Note de synthèse

## 1. Objectif du module

Savoir utiliser l’IA comme copilote pour produire des textes : articles, emails, résumés, fiches pédagogiques.  
L’enjeu : définir clairement l’objectif, le public et la structure avant de demander du contenu.

## 2. Idées clés à retenir

- Tu peux demander à l’IA de t’aider sur toutes les étapes : plan, rédaction, reformulation, synthèse.
- Plus tu es précis sur le ton (formel, convivial, expert…), mieux la réponse colle à ton besoin.
- Pour les résumés, tu peux demander différents niveaux de détail (100 / 300 / 500 mots).
- Pour la pédagogie, une bonne structure est : notion → explication simple → exemples → exercices.

## 3. Bonnes pratiques

- Toujours commencer par l’objectif du texte + le public visé (débutant, intermédiaire, expert…).
- Clarifier la structure souhaitée : titres, sous-parties, longueur approximative.
- Demander des reformulations si le ton ne te convient pas.
- Relire et adapter le texte généré : l’IA propose un brouillon, tu gardes le contrôle final.

## 4. Prompts types à garder sous la main

```plain
[Article de blog]
Je veux un article de blog sur [SUJET] pour un public [NIVEAU]. 
Structure : introduction accrocheuse, 3 à 5 parties avec titres clairs, conclusion avec appel à l’action. 
Ton : [TON]. Longueur : environ [N] mots.

[Résumés multiniveaux]
Voici un texte : [COLLER LE TEXTE].
1) Donne-moi d’abord les 5 idées clés sous forme de puces.
2) Puis un résumé en 100 mots.
3) Puis un résumé en 300 mots pour un public [NIVEAU].

[Fiche pédagogique]
Explique-moi [CONCEPT] à un public [NIVEAU]. 
Structure : définition simple, analogie, 2 exemples concrets, puis 3 questions ou exercices pour vérifier la compréhension.
```
