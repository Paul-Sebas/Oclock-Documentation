# Module 6 : Créer et maintenir son prompt book personnel -- Note de synthèse

## 1. Objectif du module

Mettre en place une vraie bibliothèque personnelle de prompts : organisée, documentée et vivante.  
Le but est de capitaliser sur tes meilleurs prompts pour gagner en efficacité au fil du temps.

## 2. Idées clés à retenir

- Un prompt book doit être organisé par catégories (créativité, analyse, communication, etc.).
- Chaque prompt mérite une petite fiche : objectif, contexte, paramètres, exemples, améliorations.
- Un bon système de tags et de notation t’aide à retrouver rapidement le bon prompt.
- Le prompt book doit évoluer : tu testes, tu améliores, tu archives ce qui marche moins bien.

## 3. Bonnes pratiques

- Choisir un outil simple pour commencer (Notion, Google Docs, GitHub, etc.).
- Garder une structure homogène pour toutes les fiches de prompts.
- Noter aussi ce qui ne fonctionne pas bien (pour éviter de refaire les mêmes erreurs).
- Prendre le réflexe de documenter dès que tu trouves un prompt vraiment efficace.

## 4. Prompts types à garder sous la main

```plain
[Créer une fiche de prompt]
Aide-moi à documenter ce prompt dans un format clair.
Prompt : [COLLER LE PROMPT].
Propose une fiche avec : objectif, contexte d’utilisation, paramètres variables, exemples de résultats, pistes d’amélioration.

[Réorganiser mon prompt book]
Voici la liste de mes prompts avec quelques notes : [LISTE].
Propose-moi une structure de catégories et de sous-catégories pour mieux les organiser, avec des tags pertinents.

[Améliorer un prompt existant]
Voici un prompt que j’utilise souvent : [PROMPT].
Propose-moi 3 variantes plus claires ou plus efficaces, en expliquant ce que tu améliores à chaque fois.
```
